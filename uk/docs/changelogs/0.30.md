# Версія 0.30
_Дата релізу: 2026-06-13_

### Карта та пошук об'єктів поруч 🗺️
- **Пошук об'єктів поруч.** Додано пошук пристроїв та ONU поблизу заданої точки: координати можна ввести вручну, обрати на карті або використати геолокацію пристрою. Результати показуються списком і на карті, з фільтром за типом і відстанню та можливістю побудувати маршрут.
- **Значне прискорення карти.** Карту суттєво оптимізовано для роботи з тисячами об'єктів: пришвидшено відмалювання маркерів, кластеризацію та підвантаження даних, додано ощадливе використання пам'яті. Завантаження великих карт стало в рази швидшим.
- Виправлено позиціонування спливаючих вікон і накладань на карті під час масштабування, додано обмеження мінімального масштабу.

### Веб-панель 💻
- **Редактор шаблонів макросів** тепер має підсвічування синтаксису та синхронне прокручування — створювати й редагувати шаблони стало зручніше.
- **Налаштування нижньої панелі навігації (мобільні пристрої):** можна вмикати/вимикати пункти та додати кнопки пошуку й «об'єктів поруч». Загальну навігацію спрощено й оптимізовано для телефонів.
- **Вибір карток інтерфейсу.** Додано керування видимістю інформаційних карток — можна приховати непотрібні панелі та прибрати зайве на екрані.
- **Покращено глобальний пошук** — зручніше модальне вікно пошуку.
- Інтерфейс налаштування **датчиків** повністю перекладено (укр./рос.).
- Підвищено стабільність роботи: усунено зайві повторні запити, покращено відображення графіків оптичних рівнів та зручність дерева ONU.

### Робота з обладнанням 🛠️
- **C-Data FD1700S (FW3) — повна підтримка.** Додано роботу з OLT C-Data FD1700S: перелік і статуси ONU, оптичні рівні, інформація про виробника та серійний номер, стан UNI-портів, VLAN, сервісні профілі, виявлення незареєстрованих ONU, описи портів, перелік PON-портів, видалення ONU.
- **C-Data FD16xxV3** — додано перегляд сервісних профілів і прив'язок ONU; профілі можна отримати навіть коли вони недоступні через SNMP.
- **ZTE C600** — додано підтримку агрегованих каналів (LACP) та отримання VLAN за портом.
- **Вільні номери ONU при реєстрації.** Тепер система показує вільні (доступні) номери ONU на кожному PON-порту, що спрощує реєстрацію нових абонентів.

### Консольні команди та макроси 🧩
Додано нові конструкції, які можна вставляти безпосередньо в текст консольної команди (у макросах та веб-консолі) для гнучкого керування виконанням:

- **`<cr>`** — надсилає після команди додатковий «Enter» (порожній рядок) і очікує на запрошення пристрою. Для команд, яким потрібне додаткове підтвердження порожнім вводом.
- **`<prompt="текст">`** — очікувати вказане приглашення/текст замість стандартного запрошення пристрою.
- **`<confirm="відповідь">`** — автоматично надіслати задану відповідь (наприклад, `y`) на запит підтвердження.
- **`<confirm-if="відповідь","текст-приглашення">`** — надіслати відповідь лише якщо пристрій показав указане приглашення (наприклад, `Are you sure?`); інакше виконання продовжується без очікування.
- **`<stream_timeout=N>`** — тимчасово збільшити час очікування відповіді (у секундах) для повільних команд; потім значення відновлюється.

Якщо конструкцію вказано з помилкою, команда повертає зрозуміле повідомлення про помилку замість зависання.

### Виправлення по обладнанню 🔧
- **Huawei** — виправлено нумерацію Gigabit Ethernet-портів (усунено конфлікти ідентифікаторів портів).
- **Визначення моделей** — зроблено надійнішим, особливо під час одночасної роботи з кількома пристроями; зрозуміліші повідомлення, коли модель не вдалося визначити.
- **C-Data FD1700S/FD17xxV3** — низка виправлень розбору даних (серійні номери ONU, індекси, описи інтерфейсів) для коректного відображення.
- **Juniper** — модулі SFP/оптики більше не завершуються помилкою, якщо потрібні дані недоступні на пристрої (повертається порожній результат).
- **Mikrotik RouterOS** — виправлено збій під час отримання DHCP-оренд без MAC-адреси.

### Бекенд, API та інтеграції ⚙️
- **Повна документація API (OpenAPI/Swagger).** Для інтеграцій додано згенеровану інтерактивну документацію всіх методів API.
- **Швидкодія та кешування** — додано кешування даних з обладнання; API та завантаження карти стали помітно швидшими.
- **Стабільніша синхронізація подій з алертами.** Події більше не «зависають» у відкритому стані після перезапуску системи сповіщень; додано «період прогріву» при старті, щоб уникнути хибних закриттів.
