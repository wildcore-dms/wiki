# Version 0.30
_Release date: 2026-06-15_

### Map & nearby search 🗺️
- **Nearby objects search.** Added the ability to find devices and ONUs near a chosen point: enter coordinates manually, pick a spot on the map, or use the device's geolocation. Results are shown as a list and on the map, with filtering by type and distance and the option to get directions.
- **Major map speed-up.** The map has been heavily optimized for working with thousands of objects: faster marker rendering, clustering, and data loading, plus lower memory usage. Loading large maps is now many times faster.
- **Objects at the same coordinates no longer cover each other.** Devices and ONUs sharing a point are now grouped and expand "dandelion"-style on click, and clustering at maximum zoom was improved — no marker is hidden underneath another.
- Fixed positioning of popups and overlays during zooming, and added a minimum zoom limit.

### Web panel 💻
- **The macro template editor** now has syntax highlighting and synchronized scrolling — creating and editing templates is more convenient.
- **Bottom navigation settings (mobile):** you can enable/disable items and add search and "nearby objects" buttons. Overall navigation was simplified and optimized for phones.
- **Choose interface cards.** Added control over the visibility of information cards — you can hide unused panels and reduce on-screen clutter.
- **Improved global search** — a more convenient search modal.
- **Macro variables from the device.** When running a macro, variables are now read directly from the device (live values), while the preview uses stored values for speed. A "Reload variables from device" button was added to the execution window.
- Fixed mass printing of QR codes.
- The **sensors** configuration UI is now fully translated (Ukrainian/Russian).
- Improved stability: removed redundant repeated requests, improved optical-level charts, and made the ONU tree easier to use.

### Equipment integration 🛠️
- **C-Data FD1700S (FW3) — full support.** Added support for C-Data FD1700S OLTs: ONU list and statuses, optical levels, vendor and serial information, UNI-port status, VLANs, service profiles, detection of unregistered ONUs, port descriptions, PON port list, and ONU deletion.
- **C-Data FD16xxV3** — added viewing of service profiles and ONU bindings; profiles can be retrieved even when they are unavailable over SNMP.
- **ZTE C600** — added support for link aggregation (LACP) and retrieval of VLANs per port.
- **Free ONU numbers during registration.** The system now shows the free (available) ONU numbers on each PON port, making it easier to register new subscribers.

### Console commands & macros 🧩
New constructs you can place directly inside a console command (in macros and the web console) to fine-tune how it runs:

- **`<cr>`** — sends an extra "Enter" (empty line) after the command and waits for the device prompt. For commands that need an additional empty-line confirmation.
- **`<prompt="text">`** — wait for the specified prompt/text instead of the device's standard prompt.
- **`<confirm="answer">`** — automatically send the given answer (e.g. `y`) in response to a confirmation request.
- **`<confirm-if="answer","prompt">`** — send the answer only if the device shows the specified prompt (e.g. `Are you sure?`); otherwise execution continues without waiting.
- **`<stream_timeout=N>`** — temporarily extend the response wait time (in seconds) for slow commands; the value is then restored.

If a construct is written incorrectly, the command returns a clear error instead of hanging.

By default, a sequence of commands (in macros and during ONU registration) now stops on the first error — this is safer and avoids running the remaining steps on a problematic device; the behavior can be changed if needed.

### Equipment fixes 🔧
- **Huawei** — fixed Gigabit Ethernet port numbering (resolved port ID collisions).
- **Model detection** — made more reliable, especially when working with multiple devices at once; clearer messages when a model cannot be detected.
- **C-Data FD1700S/FD17xxV3** — a number of data-parsing fixes (ONU serial numbers, indexes, interface descriptions) for correct display.
- **Juniper** — the SFP/optical modules no longer fail with an error when the required data is unavailable on the device (an empty result is returned).
- **Mikrotik RouterOS** — fixed a crash when retrieving DHCP leases without a MAC address.

### Backend, API & integrations ⚙️
- **Complete API documentation (OpenAPI/Swagger).** Generated interactive documentation for all API methods has been added for integrations.
- **Performance & caching** — added caching of equipment data; the API and map loading are noticeably faster.
- **More stable event-to-alert synchronization.** Events no longer get stuck in an open state after the alerting system restarts; a startup "grace period" was added to avoid false closures.

### Installation & updates ⬆️
- Added automatic retries when installing components — updates and installation are more reliable on unstable connections.
